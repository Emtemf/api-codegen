Annotation Mapping Snapshot

同一个注解扩展字段，在 Swagger 2.0 与 OpenAPI 3.0 中写法一致，生成结果也保持一致。

这张图用于 README 说明单文件输入场景。左侧和中间是两种 YAML 入口，右侧是 Maven 插件实际生成的 UserApi.java 片段与真实输出目录。

单文件 YAML 输入 → Maven 插件生成 → 统一 Controller 输出
Swagger 2.0 path 级类注解 + operation 级方法注解
Swagger 
swagger: "2.0"
info:
  title: User API
  version: "1.0"
paths:
  /users:
    x-java-class-annotations:
      - "@Secured"
      - "@AuditLog(action='USER_QUERY')"
    post:
      operationId: createUser
      x-java-method-annotations:
        - "@Permission('user:create')"
        - "@AuditLog(action='CREATE_USER')"
      responses:
        "200":
          description: OK
OpenAPI 3.0 同样使用 x-java-class / method annotations
OpenAPI 
openapi: 3.0.1
info:
  title: User API

paths:

      operationId: createUser
